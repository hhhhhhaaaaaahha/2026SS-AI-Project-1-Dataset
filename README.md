## Overview
This repository contains a historically accurate and rigorously cleaned dataset of the Chinese Professional Baseball League (CPBL) regular-season games from **2023 to 2025**. 

The dataset was primarily constructed for machine learning classification tasks, specifically predicting whether a baseball game will result in a **"Big Score" (Total Runs > 8.5)** or a **"Small Score" (Total Runs тЙд 8.5)** based on macro-level features like team matchups, ballparks, and seasonality.

---

## ЁЯЫа Data Collection Methodology

This dataset was **independently scraped from scratch** rather than sourced from existing public repositories. Data was gathered using a custom web scraper (`cpbl_crawler.py`) targeting the official CPBL schedule page. 

Collecting this data presented significant technical hurdles due to the modern CPBL website utilizing a Vue.js frontend coupled with a heavily secured ASP.NET backend. Standard `GET`/`POST` requests were actively blocked by anti-forgery tokens. To overcome this, an advanced **XHR Interception and Payload Spoofing** strategy was implemented:
1. **Headless Browser Automation:** Selenium WebDriver was deployed to naturally render the DOM and acquire legitimate session cookies.
2. **JavaScript Injection:** Custom JS was injected to hijack the native `XMLHttpRequest` prototype, intercepting the frontend's API calls before they left the browser.
3. **Payload Modification:** To bypass an API anomaly that forced future schedules (e.g., year 2026), the crawler dynamically intercepted outgoing JSON payloads, forcibly rewrote the `calendar` parameter to historical months, and captured the valid responses.

---

## ЁЯз╣ Data Cleaning & Preprocessing

The raw data (`cpbl_raw_data.csv`) contained significant noise. A rigorous cleaning pipeline (`data_cleaning.py`) was applied to prevent data leakage and model hallucination:

* **Handling "Ghost Games":** Rainouts and postponed games are frequently retained in the CPBL database with a default `0:0` score. We utilized the `PresentStatus` (or `IsGameStop`) flag to strictly filter out unplayed games.
* **Future Game Pruning:** Removed scheduled but unplayed future games that returned default values.
* **Deduplication:** Dropped exact duplicates generated by the API's pagination and bulk-return behavior using a composite key (`GameDate`, `VisitingTeamName`, `HomeTeamName`).
* **Feature Engineering:** Extracted `Month` and `DayOfWeek` from the game date to capture seasonality effects (e.g., hitter-friendly summer months).
* **Target Creation:** Summed the home and visiting scores, thresholding at `8.5` to create the binary target label `IsBigScoringGame`. Original score columns were dropped to prevent data leakage.

**Final Dataset Statistics:**
* Total verified records: **1,128 games**
* Time span: **2023 - 2025**

## ЁЯУК Data Dictionary

The final machine learning-ready dataset (`cpbl_ml_ready_dataset.csv`) utilizes One-Hot Encoding for categorical features. Below is the description of the core features before/after transformation:

| Feature Name | Data Type | Description |
| :--- | :--- | :--- |
| `Month` | Integer (4-11) | The month the game was played (captures seasonality/weather). |
| `DayOfWeek` | Integer (0-6) | Day of the week (0 = Monday, 6 = Sunday). |
| `FieldAbbe_*` | Binary (0/1) | One-hot encoded ballpark (e.g., `FieldAbbe_хдйцпН`, `FieldAbbe_ц┤▓щЪЫ`). Captures park factors. |
| `VisitingTeamName_*` | Binary (0/1) | One-hot encoded visiting team (e.g., `VisitingTeamName_ф╕нф┐бхЕДх╝Я`). |
| `HomeTeamName_*` | Binary (0/1) | One-hot encoded home team (e.g., `HomeTeamName_хС│хЕищ╛Н`). |
| **`IsBigScoringGame`** | **Integer (0/1)** | **Target Label.** `1` if Total Runs > 8.5 (Big Score), `0` otherwise. |
